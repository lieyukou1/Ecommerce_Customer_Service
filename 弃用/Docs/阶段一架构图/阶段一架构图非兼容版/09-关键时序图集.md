# 09-关键时序图集

## 这册看什么

这一册只画关键时序，不再解释类结构。

## 图 1：文本消息 - 工单状态查询

```mermaid
sequenceDiagram
    participant U as 住户
    participant FE as Frontend
    participant API as chat_router
    participant Service as DialogueService
    participant Engine as DialogueEngine
    participant Planner as TurnPlanner
    participant Validator as TurnPlanValidator
    participant Task as TaskHandler
    participant ESB as 物业中台

    U->>FE: 文本询问工单状态
    FE->>API: POST /api/chat
    API->>Service: process_message(UserMessage)
    Service->>Engine: hand_dialogue(state, user_message)
    Engine->>Planner: predict(...)
    Planner-->>Engine: TurnPlan(task)
    Engine->>Validator: validate(...)
    Validator-->>Engine: valid=True
    Engine->>Task: handle() [占位]
    Task-->>Engine: 后续应生成查询动作
    Engine-->>Service: ProcessResult
    Service-->>API: ProcessResult
    API-->>FE: ChatResponse
```

## 图 2：文本消息 - 工单进度查询

```mermaid
sequenceDiagram
    participant U as 住户
    participant Engine as DialogueEngine
    participant Planner as TurnPlanner
    participant Validator as TurnPlanValidator
    participant Task as TaskHandler

    U->>Engine: text("这条工单处理到哪一步了")
    Engine->>Planner: predict(...)
    Planner-->>Engine: TurnPlan(task.progress)
    Engine->>Validator: validate(...)
    Validator-->>Engine: valid=True
    Engine->>Task: handle() [占位]
    Note over Task: 后续应承接 /work-orders/{id}/progress
```

## 图 3：对象消息 - 选中工单后继续追问

```mermaid
sequenceDiagram
    participant FE as Frontend
    participant Engine as DialogueEngine
    participant State as DialogueState
    participant Task as TaskHandler

    FE->>Engine: object(type=work_order, id=...)
    Engine->>State: set_focused_object(...)
    Engine->>Engine: _resolve_object_commands(...)
    alt 缺少 work_order_id 槽位
        Engine->>Task: handle() [占位]
        Note over Task: 后续应执行 SetSlotsCommand
    else 已有活跃任务
        Engine->>Task: handle() [占位]
    else 无任务上下文
        Engine->>Engine: clarify for intent
    end
```

## 图 4：澄清分支 - 校验失败到用户补充

```mermaid
sequenceDiagram
    participant U as 住户
    participant Engine as DialogueEngine
    participant Validator as TurnPlanValidator
    participant Clarify as ClarifyResponder

    U->>Engine: 模糊 / 不完整表达
    Engine->>Validator: validate(...)
    Validator-->>Engine: valid=False, reason=...
    Engine->>Clarify: respond(state, reason)
    Clarify-->>Engine: BotMessage(澄清提示)
    Engine-->>U: 请补充对象 / 意图 / 任务方向
    U->>Engine: 补充说明
```

## 一句话结论

当前已经比较清楚的是“决策链时序”，真正待补的是 task 轨把业务执行串到物业中台的那一截。
