# 7B 部署现场进度备忘

## 1. 本次目标

在智川云提供的 GPU 容器环境中，为当前物业智能管家项目部署一套可通过 OpenAI 兼容接口访问的 7B 本地模型服务，优先方案为：

- 模型：`Qwen/Qwen2.5-7B-Instruct`
- 服务：`vLLM`
- 接口：`/v1/chat/completions`

## 2. 当前环境判断

本次不是传统云服务器 / VM，而是平台托管的 GPU 容器环境。

已确认信息：

- 运行身份：`root`
- Python：`3.12.3`
- GPU：`NVIDIA GeForce RTX 4090`
- 显存：`49140 MiB`
- Docker：不可用，`docker: command not found`
- 结论：**不走 Docker，改为容器内直接安装并启动 vLLM**

## 3. 存储情况

现场检查结果：

- 系统盘 `/`：`30G`，剩余约 `25G`
- 数据盘 `/root/rivermind-data`：`49G`，剩余约 `41G`

结论：

- 当前空间足够继续做 7B 首版推理部署
- 大文件统一放数据盘，避免系统盘被 Hugging Face 缓存和临时文件打满

## 5. 已定位问题

### 5.1 直接拉取 Hugging Face 失败

`vLLM` 首次启动失败的直接原因已明确：

```text
[Errno 101] Network is unreachable
HEAD https://huggingface.co/Qwen/Qwen2.5-7B-Instruct/resolve/main/config.json
```

结论：

- 容器不能直接访问 `huggingface.co`
- 所以后续错误如 `Can't load the configuration` 都只是连锁反应

### 5.2 镜像方案可用

已验证在当前 shell 中设置镜像后，可以下载模型文件：

```bash
export HF_ENDPOINT=https://hf-mirror.com
```

并已成功执行：

```python
from huggingface_hub import hf_hub_download
path = hf_hub_download(
    repo_id="Qwen/Qwen2.5-7B-Instruct",
    filename="config.json",
    cache_dir="/root/rivermind-data/hf-cache"
)
print(path)
```

结果：

- `config.json` 已成功下载
- 说明当前容器并非完全断网
- 后续应改为：**先完整下载模型到本地，再用本地路径启动 vLLM**
