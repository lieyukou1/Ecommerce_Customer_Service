继续按第三阶段排期推进两件事：

2. 把状态机里的观测表达继续往外剥

## 为什么现在做

前面虽然已经把目录和职责拆开了，但还留着两个结构性问题：

1. `focus/text_switch.py` 仍然是“打分挑最像”
2. `runtime/state_machine.py` 仍然同时负责：
   - 状态迁移
   - task action 描述
   - task outcome 推断

这两个点都会继续把架构往“看起来拆了，实则还揉在一起”拉回去。

## 实际改动

1. 新增 `runtime/observation.py`
   - `RuntimeObservationBuilder`
   - `TaskTransitionOutcome`
2. 调整 `runtime/state_machine.py`
   - `finalize_task_transition()` 现在直接接收 `outcome`
   - 不再负责拼接 task transition 描述
   - 不再负责推断 task outcome
3. 调整 `dialogue_engine.py`
   - 由 `RuntimeObservationBuilder` 负责生成 object reason 和 task outcome
4. 调整 `turns/decision_protocol.py`
   - task / knowledge route reason 改由 `RuntimeObservationBuilder` 生成
5. 重写 `focus/text_switch.py`
   - 去掉“打分挑最高”
   - 改成三段式唯一强命中：
     - `id` 唯一命中
     - 标题精确唯一命中
     - 同类型前提下的标题包含唯一命中
