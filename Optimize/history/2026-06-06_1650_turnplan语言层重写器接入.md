# 2026-06-06 16:50 TurnPlan 语言层重写器接入

## 本轮目标

- 不再继续靠模板拼接修语言层
- 改为引入基于外部 LLM 的“自然化重写器”
- 保留原有 schema、状态字段、label、flow / intent 体系，只替换语言表达

## 为什么现在做

- 用户人工抽检后明确指出：当前数据集的问题不再是 coverage，而是生成范式本身太像脚本
- 典型问题包括：
  - 模板拼接痕迹明显
  - 用户自我标注意图
  - 语言过于理想化，默认用户总能精准命中标签
- 这些问题继续靠模板修，只会越补越僵，必须换到“label 保留、语言整层重写”的路线

## 实际改动

1. 新增脚本：
   - `Material/Datasets/turnplan-phase1/scripts/llm_rewrite_dataset.py`
2. 设计了新的重写 prompt，重点约束：
   - 替换式重写，不是在原句上打补丁
   - 去除模板拼接腔
   - 去除“我在补槽/这是投诉原因补充”这类标注腔
   - 允许自然模糊、省略、情绪化表达
   - 让 slot 值藏在真实表达里，而不是用显式标记词
3. 给不同 bucket 增加了专项提示：
   - `all_null`
   - `directive_exit_runtime`
   - `active_task_slot_fill`
   - `task_interrupt_resume_cancel`
   - `work_order_read_only_task`
   - `work_order_business_urge / complaint`
4. 改造了配套脚本，使其支持指定输入目录：
   - `validate_dataset.py --input-dir`
   - `audit_dataset.py --input-dir --report-dir`
   - `export_sft.py --input-dir --output-dir`
5. 后续根据人工审计再次收紧：
   - 把模板拼接反例、意图自我标注反例、系统内部视角反例直接写进 prompt
   - 加入启发式质量门禁，拦截明显的“我在补槽 / 别默认进入办理 / 按这单本身回答”这类句式
6. 升级成多模型重写器：
   - 自动加载用户提供网关模型
   - 自动加载项目现有 DeepSeek 配置
   - 每条样本按稳定路由选择重写模型
   - 尽量由另一模型交叉复核
   - 调用前先做模型探活，失败时自动回退可用模型
