# 当前功能面基线说明

## 定位

这是当前物业版智能管家项目的功能面基线说明。
它用于说明现阶段哪些链路已可联测，哪些只适合做占位验证，哪些暂时不纳入功能闭环。

## 什么时候看

- 准备跑功能测试脚本前
- 判断某个失败是“当前未实现”还是“新增回归问题”时
- 新增测试场景前

## 解决什么问题

- 明确测试边界
- 避免把占位能力误判成回归故障
- 为后续增量测试扩面提供基线

## 不用于什么

- 不代替实现计划
- 不代替代码校验报告
- 不描述老师参考代码，只描述当前正式项目

## 当前结论

当前项目已进入“可验证主干、尚未补完能力”的状态：

- 已可联测
  - 前端住户对象列表加载
  - 前端文本消息发送
  - 前端工单对象发送
  - 前端服务项目对象发送
  - 本地 `/api/chat` 主链请求收发
  - 状态装配、规划链、澄清链

- 仅可占位验证
  - task 轨最终执行
  - 工单状态/进度后的真实结果回复

- 暂不纳入闭环
  - knowledge 轨
  - chitchat 轨
  - `/api/chat/history`
  - 数据库状态断点校验
  - 虚拟机侧深层服务日志回捞

## 当前已知不对齐项

1. 前端请求字段仍是 `sender_id`，本地智能管家后端主口径是 `resident_id`
2. 前端会请求 `/api/chat/history`，本地智能管家后端当前未提供对应能力
3. 前端默认代理口径仍更贴近物业中台，而不是本地智能管家 `18082`

## 测试口径

- “硬通过”：HTTP 成功、页面无前端异常、返回至少一条 bot message
- “可接受占位”：链路完整且无异常，但返回当前已知占位结果
- “硬失败”：5xx、JS 运行错误、请求未发出、响应结构不合法、页面无反馈
